# Як це працює

## Суть

Платформа дає людині стратегічний контроль над метою, а виконання автоматизує система.  
Надійність і прозорість забезпечує Git як єдине джерело істини.  
Виконання координується через `agent-server`, сесії, події та спеціалізовані поверхні.  
Система підтримує контроль користувача, багатомовність, супровід і самопокращення через ретроспективний аналіз.

## Стратегічне управління

Людина визначає мету та зберігає контроль через критичні гейти системи.  
Платформа бере на себе керування виконанням і підтримує самовдосконалення через ретроспективний аналіз.  
Фокус — на прозорості, контролі користувача та архітектурній стійкості.

## Ядро системи

Git використовується як єдине джерело істини для постійного зберігання, надійності та аудиторського сліду.  
Relay відповідає за ефемерну координацію.  
Ядро реалізує рекурсивний граф задач із динамічним розкладом і атомарним керуванням життєвим циклом завдань.

## Виконання

Централізований `agent-server` оркеструє інтерактивні сесії через протокол подій `Envelope`.  
Система підтримує різні режими роботи та міграцію сесій між хостами.  
Поверхні (`surfaces`) налаштовують контекст і інструменти під конкретне завдання.

## Технологічна й експлуатаційна основа

Nitra MT має стандартизований технологічний стек: бізнес-логіка інкапсульована в `@7n/mt` на Bun/JS, а хост реалізований у Rust.  
Огляд також охоплює взаємодію учасників і пристроїв, механізми багатомовності, цикл самопокращення та контракти експлуатації.
