# Виконання: хости, сесії, поверхні

## Суть

Система функціонує через централізований `agent-server`, який оркеструє та керує інтерактивними сесіями через протокол подій (`Envelope`). Архітектура підтримує режими роботи та міграцію сесій між хостами. Робота агента визначається спеціалізованими "поверхнями" (`surfaces`), які налаштовують контекст і інструменти під конкретне завдання. Технологічний стек Nitra MT стандартизований, де бізнес-логіка інкапсульована в `@7n/mt` (Bun/JS), а хост реалізований у Rust.

## Runtime: хости, сесії, поверхні

Централізований `agent-server` відповідає за оркестрацію та управління інтерактивними сесіями. Він забезпечує єдиний стан для клієнтів, взаємодіючи через протокол подій (`Envelope`). Підтримуються різні режими роботи та механізми міграції сесій між хостами.

## Спеціалізовані поверхні (surfaces)

Система використовує різні профілі (`surfaces`) для оптимізації AI-агента під конкретне завдання (наприклад, дизайн чи командний рядок). Кожна поверхня налаштовує контекст, дозволені інструменти та системи промптів, що знижує навантаження на контекст.

## Nitra MT — референсний стек

Логічний контракт графа інкапсульовано у `@7n/mt` (Bun/JS), що забезпечує єдність бізнес-правил. Довгоживучий хост-процес реалізовано на Rust, який використовує цей контракт через підпроцеси. Вся інфраструктура стандартизована навколо OpenAI-сумісного транспорту.
