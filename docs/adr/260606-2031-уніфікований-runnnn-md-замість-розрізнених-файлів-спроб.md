## ADR Уніфікований `run_NNN.md` замість розрізнених файлів спроб

## Context and Problem Statement
Система мала окремі файли для різних акторів і результатів: `repair_history.md` (append-only журнал інженера), `error.md` (append-only журнал збоїв), `outputs.md` (append-only результати). Це суперечило принципу "всі файли під час роботи immutable" і плодило різні структури для однотипних операцій.

## Considered Options
* Зберегти окремі `repair_history.md`, `error.md`, `outputs.md` (append-only)
* Уніфікований `run_NNN.md` — один immutable-файл на одну спробу будь-якого актора

## Decision Outcome
Chosen option: "Уніфікований `run_NNN.md`", because всі спроби (агент, інженер, людина) структурно ідентичні: є намір (`## Reasoning`), є технічний результат (`## Script`), є посилання на продукт (`## Ref`). Єдина схема усуває append-only виключення і робить граф станів повністю file-existence-based.

### Consequences
* Good, because transcript фіксує очікувану користь: стан вузла повністю визначається наявністю `run_*.md` і `outputs_*.md` без читання вмісту.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Схема `run_NNN.md`: `created_at`, `actor: agent|engineer|human`, `result: success|failed`, `worktree:` (якщо failed); секції `## Reasoning` (обов'язкова), `## Script` (опційна, wrapper), `## Ref` (опційна). Файл `/Users/vitaliytv/www/nitra/cursor/npm/docs/mt.md`, секція `### run_NNN.md`.

## ADR Автоматичний тригер наступника через git post-merge hook

* Людина вручну запускає кожен вузол після merge
* `mt run` як довготривалий демон-оркестратор

Chosen option: "Git post-merge hook → `mt run --auto`", because кожен компонент виконує одну функцію: wrapper запускає агента і мержить при успіху, git-інфраструктура тригерить продовження, `mt run --auto` сканує і запускає всіх розблокованих наступників.

* Good, because transcript фіксує: система "жива" — автоматично рухається до кінця після старту кореневого вузла.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
