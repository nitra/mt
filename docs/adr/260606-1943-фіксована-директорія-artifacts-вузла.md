description: Великі файли-артефакти вузла зберігаються у фіксованій піддиректорії artifacts/, а ref з outputs_NNN.md вказує всередину неї.

**Status:** Accepted
**Date:** 2026-06-06

## Context and Problem Statement

У файловій структурі вузла потрібно визначити, де зберігати великі файли-артефакти, на які посилається `outputs_NNN.md`. У transcript обговорювалися два підходи: дозволити `ref:` вказувати на будь-який шлях або зафіксувати одну директорію артефактів у межах вузла.

## Considered Options

- `ref:` може вказувати на будь-який шлях.
- `tasks/<node>/artifacts/` як фіксована директорія для великих файлів вузла.

## Decision Outcome

Chosen option: "`tasks/<node>/artifacts/` як фіксована директорія для великих файлів вузла", because варіант з фіксованою директорією дає порядок: усі артефакти вузла лежать в одному місці, а `ref:` в `outputs_NNN.md` вказує всередину неї.

### Consequences

- Good, because усі великі файли-артефакти вузла зберігаються в одному передбачуваному місці.
- Good, because `outputs_NNN.md` може посилатися на артефакти через `ref:` без розкидання файлів по довільних шляхах.
- Bad, because transcript не містить підтвердження негативних наслідків фіксованої директорії `artifacts/`.
- Neutral, because transcript фіксує лише зміну документа `npm/docs/mt.md`, а не реалізацію CLI.

## More Information

Зафіксована структура вузла:

```text
tasks/<node-id>/
  task.md
  invalidated
  run_001.md
  outputs_001.md
  artifacts/
    anomalies.json
    report.md
```

Файл документації, який змінювався в transcript: `npm/docs/mt.md`.

Transcript facts:

- Попередній текст: `artifacts/` не визначено — агент вільно вказує `ref:` на будь-який шлях.
- Новий текст: `artifacts/` → `tasks/<node>/artifacts/` — фіксована директорія для великих файлів вузла; `ref:` в `outputs_NNN.md` завжди вказує всередину неї.
