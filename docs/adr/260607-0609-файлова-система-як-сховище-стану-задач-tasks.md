## ADR Файлова система як сховище стану задач (tasks/)

## Context and Problem Statement
У проєкті з'явилась потреба відстежувати кілька пов'язаних робочих одиниць (UI-перегляд, тестування скілу, міграція оркестратора). Потрібна конкретна структура для зберігання і читання стану кожної задачі без централізованої бази даних.

## Considered Options
* Файлова система: один каталог на вузол (`tasks/<name>/`), стан — через наявність файлів
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Файлова система: один каталог на вузол", because структура вже описана в `npm/docs/mt.md` як «Рекурсивний складений ОАГ» і прийнята як базова архітектура системи задач.

### Consequences
* Good, because transcript фіксує очікувану користь: стан `waiting` визначається лише наявністю `task.md` (без `run_*.md` і `outputs_*.md`), що не потребує зовнішнього сховища.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

## ADR Формат task.md: YAML frontmatter + секції Task / Done when / Inputs

## Context and Problem Statement
Кожен вузол задачі потребує машинозчитуваних метаданих (час створення, бюджет, залежності) та людинозчитуваного опису умов завершення і вхідних даних.

## Considered Options
* YAML frontmatter (`created_at`, `budget_sec`, `parent`, `deps`) + markdown-секції `## Task`, `## Done when`, `## Inputs`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "YAML frontmatter + markdown-секції", because формат взятий безпосередньо з `npm/docs/mt.md` і застосований однаково до всіх трьох створених задач.

### Consequences
* Good, because transcript фіксує очікувану користь: секція `## Done when` задає чіткі верифікаційні критерії (наприклад, «real mutant знищено, тести зелені, tier-routing працює»), а `budget_sec` обмежує ресурс виконання.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Приклад значень `budget_sec`: `ui-task-view` — 3600 с, `coverage-skill-test` — 1800 с, `skills-orchestrator-migration` — 7200 с. Назви каталогів вузлів — kebab-case англійською, вміст `task.md` — українською.
