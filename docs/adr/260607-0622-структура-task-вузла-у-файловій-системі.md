## ADR SSH gateway з backend-контролем доступу до k8s замість kubectl

## Context and Problem Statement

## Considered Options
* `kubectl port-forward` + SSH у поді (Variant 1) — відхилено

## Decision Outcome

### Consequences
* Good, because transcript фіксує очікувану користь: доступ до k8s dev-середовища без передачі `kubectl` credentials розробникам; централізований контроль через backend.
