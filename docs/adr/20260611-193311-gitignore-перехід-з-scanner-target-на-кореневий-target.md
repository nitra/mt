## ADR `.gitignore` — перехід з `scanner/target/` на кореневий `target/`

## Context and Problem Statement
У `.gitignore` була прописана точкова директорія `scanner/target/`, але кореневий `Cargo.toml` визначає Cargo workspace (`members = ["scanner"]`), тому Rust складає артефакти в кореневу директорію `target/`, а не в `scanner/target/`. Через це коренева `target/` потрапляла в невідстежувані файли (`?? target/`).

## Considered Options
* Залишити `scanner/target/` — не покриває реальний шлях збірки workspace.
* Замінити на `target/` на рівні кореня — покриває весь Cargo workspace output.

## Decision Outcome
Chosen option: "Замінити на `target/`", because Cargo workspace завжди виводить артефакти в кореневий `target/`; правило `scanner/target/` було хибно-специфічним і не ігнорувало реальну директорію збірки.

### Consequences
* Good, because кореневий `target/` тепер ігнорується і не з'являється у `git status` як невідстежуваний файл.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Файл: `.gitignore`
- Конфіг workspace: `Cargo.toml` (root) — `[workspace] members = ["scanner"]`
- Зміна: `scanner/target/` → `target/` (рядок 5)
