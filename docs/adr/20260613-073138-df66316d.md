## Фаза 1 — Розширити Rust-бінарник

Поточна `derive_node_state` у `scanner/src/lib.rs` — **заглушка**, повертає `Draft`. Треба портувати повну 10-крокову машину стану з `npm/lib/core/state.mjs:deriveNodeState`.

**Нові поля `TaskNode`:**
```rust
pub dir: String,           // абс. шлях = tasks_dir + "/" + path
pub children: Vec<String>, // змінити з Vec<TaskNode> на Vec<String> (id)
pub body: String,          // вміст task.md після frontmatter
```

**Зміни в логіці `scan()`:**
1. Читати `tasks_dir/../.mt.json` → `agent_retry_max` (дефолт 3).
2. Читати `task.md` → strip frontmatter → `body`.
3. `derive_node_state` — 10 кроків у пріоритеті:
- `pending-audit_NNN.md` без `audit-result` або `result≠success` → `pending_audit`
- `resolved` (fact + audit success) → `resolved`
- `unresolvable.md` → `unresolvable`
- `running_<pid>_until_` маркер **або** активний worktree (`git worktree list --porcelain` через `std::process::Command`) → `running`
- composite plan-review / spawned (читати frontmatter `plan_NNN.md`)
- `a.md` + `failed_streak ≥ agent_retry_max` → `failed`
- `a.md` + streak < max → `waiting`
- `h.md` → `pending`
- інакше → `unassigned`
4. Другий прохід — `blocked`: вузол переходить у `blocked` якщо хоча б одна його залежність не в `{done, cancelled, resolved, unresolvable}`.
5. `children` — `Vec<String>` (id дочірніх задач, не вкладені об'єкти).

## Фаза 2 — `scanner.mjs` → тонкий шим

Файл **повністю переписується**: нуль ФС-обходу, нуль `deriveNodeState`.

```js
import { spawnSync } from 'node:child_process'
import { resolveScannerBin } from './scanner-bin.mjs'

function adaptNode(n) {
return {
id: n.id, dir: n.dir, path: n.path,
composite: n.is_composite,            // is_composite → composite
children: n.children,                 // вже string[]
deps: n.deps, body: n.body,
agentRetryMax: n.agent_retry_max,     // snake → camel
state: n.state.replace(/_/g, '-'),    // snake_case → kebab-case
}
}

export async function scanTasks(mtDir) {
const r = spawnSync(resolveScannerBin(), ['scan', mtDir], { encoding: 'utf8' })
if (r.status !== 0) throw new Error(`mt-scanner failed:\n${r.stderr}`)
return JSON.parse(r.stdout).map(adaptNode)
}

export function findTasks(mtDir) {
const r = spawnSync(resolveScannerBin(), ['scan', mtDir], { encoding: 'utf8' })
return r.status !== 0 ? [] : JSON.parse(r.stdout).map(n => join(mtDir, n.path))
}
```

`topoSort`, `areDepsResolved`, `getActiveWorktrees`, `parseWorktreeList` — **лишаються в JS** без змін.

`scanTasks` більше **не приймає** `fs` та `config` — DI знята.
