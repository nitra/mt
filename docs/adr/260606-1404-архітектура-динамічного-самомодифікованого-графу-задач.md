## ADR Архітектура динамічного самомодифікованого графу задач

## Context and Problem Statement
Необхідно задокументувати архітектуру системи, де задачі розбиваються на динамічний граф виконання агентами, файлова система слугує state store, а агенти-інженери автономно відновлюють невдалі вузли. Сесія охоплює еволюцію від початкового концепту до детального file-system контракту.

## Considered Options
* Рекурсивний складений ОАГ з file-based state store і git worktree як межою ізоляції
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Рекурсивний складений ОАГ з file-based state store", because це забезпечує безкоштовну персистентність, LLM-friendly формат (Markdown+YAML), атомарність через git worktree merge, і природний аудит-трейл без додаткової інфраструктури.

### Consequences
* Good, because transcript фіксує очікувану користь: worktree merge як єдина точка рішення оркестратора усуває race conditions без будь-якого механізму блокування.
* Bad, because transcript не містить підтверджених негативних наслідків.
