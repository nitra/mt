type: ADR
title: Read-only UI для моніторингу графу mt
description: UI для mt має бути read-only переглядачем станів графу з polling-оновленням через файловий scan, а керування лишається в CLI.
---

**Status:** Accepted
**Date:** 2026-06-07

## Context and Problem Statement

Архітектура `npm/docs/mt.md` описує файловий стан графу, де людині потрібні два основні режими роботи: щоденний моніторинг станів вузлів і розслідування failed або invalidated вузлів. Потрібно визначити роль UI відносно CLI та файлового контракту `tasks/`.

## Considered Options

- Read-only UI як observability tool, а всі керуючі дії виконуються через CLI.
- UI з кнопками `kill` / `run` та редагуванням `task.md`.
- Polling через `mt scan --json`.
- SSE або real-time stream логів.
- Tree-view за файловою ієрархією вузлів.
- DAG-граф зі стрілками залежностей.

## Decision Outcome

Chosen option: "Read-only UI з polling через `mt scan --json` і tree-view", because стан вузла визначається файлами, UI потрібен для спостереження й розслідування, а control plane лишається CLI-first.

### Consequences

- Good, because UI прямо відображає контракт моніторингу з `npm/docs/mt.md`: scan файлів, дерево станів, failed-вузли та активні worktree.
- Good, because tree-view відповідає фізичній структурі вкладених директорій `tasks/` і простіший за DAG-rendering.
- Bad, because transcript не містить підтвердження негативних наслідків для read-only UI.
- Neutral, because real-time stream логів не підтримується: агент пише `run_NNN.md` після завершення, а не потокові логи.

## More Information

UI flow:

- Щоденний моніторинг: відкрити UI → побачити дерево вузлів зі станами `waiting`, `running`, `resolved`, `failed`, `invalidated` → автооновлення polling через `mt scan --json`.
- Розслідування інциденту: клік на failed/invalidated вузол → панель деталей з `task.md` і `run_NNN.md` → людина читає `## Reasoning` → діє через CLI.
- Прогрес задачі: розгорнути кореневий вузол → побачити дочірні вузли та їхні стани.

Не входить у UI: кнопки `kill` / `run`, редагування `task.md`, повний inline-вміст великих `outputs_NNN.md`, real-time stream логів агента.
