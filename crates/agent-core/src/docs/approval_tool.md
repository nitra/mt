## Огляд

Механізм надає захист для деструктивних `ToolCall`s, забезпечуючи їхнє виконання лише після асинхронного підтвердження від авторизуючої системи. Декоратор `GatedTool` перехоплює виклик `Tool` перед його виконанням, ініціюючи `ApprovalRequest` у кімнату та очікуючи верифікованого вердикту з таймаутом. У разі відмови чи перевищення часу, виклик тула не виконується, а агент отримує причину в історії через `ToolOutput::failure`, що дозволяє йому адаптуватися, не припиняючи роботу. Публічні інтерфейси включають `ApprovalRequester` та `gate_tools`.

## Поведінка

ApprovalRequester — Визначає асинхронний механізм для отримання вердикту на виконання дії, який реалізує хост-система.
gate_tools — Обгортає і реєструє визначений набір тулів у реєстр, застосовуючи до них механізм схвалення перед їхнім викликом.

## Публічний API

ApprovalRequester — Асинхронна спроба отримати схвалення вердикту, яке завершується або схваленням, або тайм-аутом.
gate_tools — Доступ до інструментів реєстратора, які дозволені політикаю схвалення.
