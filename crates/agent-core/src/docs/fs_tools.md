## Огляд

Цей модуль надає інтерфейс для взаємодії агента з файловою системою у sandbox-середовищі. Він ініціалізує набір тулів через публічну функцію `register_workspace_tools`, що дозволяє агенту виконувати операції з файлами (зчитування, створення, модифікація, перелік) з суворо обмеженими шляхами — лише відносними до кореня робочої директорії. Усі операції обробляються з механізмом виявлення помилок (`fail-safe`), що означає, що функції повертатимуть порожнє значення замість викидання винятків.

## Поведінка

1. Викликати `register_workspace_tools` для ініціалізації набору файлових тулів.
2. Передати існуючий `ToolRegistry` та абсолютний шлях до кореня робочої директорії (`root`).
3. Система інкапсулює кореневий шлях.
4. Реєструються три основні тули, що працюють з файловою системою в межах встановлених меж безпеки:
    - `read_file` для отримання вмісту текстового файлу.
    - `write_file` для створення або перезапису файлу.
    - `list_files` для переліку вмісту директорії.
5. Всі шляхові операції виконуються виключно відносно кореня робочої директорії; абсолютні шляхи або спроби виходу за межі кореня (`..`) відхиляються.

## Публічний API

register_workspace_tools — Дозволяє системі знати про файлові інструменти, доступні у робочій області, запущеній із `root`.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
