## Огляд

Модуль надає уніфікований інтерфейс для взаємодії з мовними моделями (LLM), емулюючи функціонал OpenAI-compatible Chat Completions. Він слугує посередником, дозволяючи системі звертатися до різноманітних LLM-провайдерів — як локальних (наприклад, Ollama, LM Studio), так і хмарних (через LiteLLM-профілі). Конфігурація забезпечується через `base_url`, який є мінімальним спільним знаменником для таких середовищ, як <http://127.0.0.1:8080/v1>. Модуль реалізує логіку fail-safe, перехоплюючи помилки та повертаючи порожні значення замість викидання винятків.

## Поведінка

Поведінка:
OpenAiProvider створює обгортку для взаємодії з LLM-транспортом, що сумісний з OpenAI через `async-openai`.
new ініціалізує обгортку, конфігуруючи базовий URL API (наприклад, <http://127.0.0.1:8080/v1>) та ключ API для підключення до зовнішнього LLM-сервісу.

## Публічний API

OpenAiProvider — надає функціонал, сумісний з OpenAI, використовуючи бібліотеку `async-openai`.
new — дозволяє вказати базовий URL API (наприклад, <http://127.0.0.1:8080/v1>) та токен доступу.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
