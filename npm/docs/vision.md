# Мета: платформа задач для людей і ШІ

> Нормативний документ «навіщо». Архітектура ([architecture/](architecture/index.md)) — це «як»; кожне архітектурне рішення має підтримувати цю мету.

## Теза

Для управління проєктами/задачами створено багато прекрасних продуктів — від Trello до Jira. Але всі вони проєктувались у світі, де **єдиними виконавцями були люди**.

Зараз ШІ стає **самостійною одиницею**, яка вирішує ряд задач — і деякі набагато краще за людей. Тому потрібна платформа, де у проєктах і задачах працюють не тільки люди, а здебільшого саме ШІ — **як рівноправні виконавці, з тим самим контрактом задачі**.

Git/GitHub — це насамперед інструмент, спеціалізований для розробників. Для нового світу він лишається **субстратом** (надійність, офлайн, audit trail, відсутність lock-in), але не інтерфейсом: зручність роботи людей і машин має бути понад ним.

## П'ять крос-вимірів

1. **Крос-виконавча.** Людина і агент — взаємозамінні виконавці одного вузла задачі; той самий життєвий цикл, бюджети, аудит, ескалація.
2. **Крос-девайсова.** Мак / сервер / телефон — рівноправні точки участі: виконання там, де є ресурси; контроль і approve — з будь-якого пристрою, включно без git-доступу.
3. **Крос-програмкова.** Не один універсальний Codex/Claude, а **спеціалізовані тули** (designer, writer, cli, …), які працюють у рамках певної задачі/процесу — через профілі поверхонь, skills і провайдерів на вузол.
4. **Крос-людська.** Кілька людей у одній задачі: membership, ролі, запрошення, мультипартійні підписані approvals.
5. **Крос-мовна.** Раніше все приводилось до англійської — незручно для не носіїв, але багатомовність була «дорогою». Тепер ШІ робить контент багатомовним автоматично: канонічний вміст — мовою автора, поверхня рендерить кожному учаснику його мовою. Це сприяє крос-національній взаємодії.

## Наслідки для фокусу

- Пріоритет — ядро графа + одна поверхня, доведені **dogfooding-ом**: розробка MT ведеться через MT.
- Перший продукт — «mission control для агентів на всіх пристроях» (мак + сервер + телефон-approver); масштаб «заміни Jira» (команди, крос-мовність) — наступне кільце.
- Git лишається Шаром 0 (істина), але жоден сценарій для не-розробника не має вимагати роботи з git напряму.
