# Як це працює

## Суть

Платформа забезпечує універсальний контракт для виконання завдань, що гарантує роботу незалежно від пристрою чи мови. Система будується на архітектурі з чітко визначеними шарами, де Git слугує єдиним джерелом істини. Централізований агент-сервер оркеструє сесії, а спеціалізовані "поверхні" визначають контекст для роботи.

## Ключові аспекти

### Контракти та універсальність

Платформа використовує універсальний контракт задачі, що забезпечує стабільну роботу незалежно від середовища чи мови. Це дозволяє реалізувати системний аудит та багатомовну підтримку в ядрі для самооптимізації.

### Ядро та керування станом

Функціональність базується на рекурсивному графі задач, який динамічно розкладується. Git використовується як надійне джерело істини для забезпечення прозорого аудиторського сліду, розділяючи постійне зберігання та ефемерну координацію.

### Оркестрація та виконання

Робота здійснюється через `agent-server`, який керує інтерактивними сесіями за допомогою протоколу подій (`Envelope`). Контекст завдання налаштовується через спеціалізовані "поверхні", а бізнес-логіка інкапсульована у стандартизованому стеку Nitra MT (Bun/JS).

### Прозорість та доступ

Система забезпечує високий рівень прозорості, підтримуючи взаємодію між різними пристроями та учасниками. Це включає механізми багатомовності та цикл самопокращення, що відповідає контрактам експлуатації.
