# Ядро: граф задач і координація через git

## Суть

Система базується на Git як єдиному джерелі істини для забезпечення надійності та прозорого аудиторського сліду. Архітектура розмежовує відповідальність між шарами, де Git забезпечує постійне зберігання, а Relay – ефемерну координацію. Ядро реалізує рекурсивний граф задач із динамічним розкладом, використовуючи механізми для атомарного керування життєвим циклом завдань.

## Рішення злиття і шари системи

Архітектурний опис визначає загальну структуру системи, де Git слугує основним джерелом істини. Система використовує криптографічні підписи для гарантування надійності, зводячи складні операції до атомних транзакцій. Це забезпечує високу стійкість та прозорий аудиторський слід для всіх рішень.

## Ядро: рекурсивний граф задач

Цей документ описує архітектурну модель рекурсивного графа задач з динамічним розкладом. Він визначає чіткий контракт для вузлів (атомарних або складених), забезпечуючи інкапсуляцію. Багаторівнева система станів керує життєвим циклом задач, гарантуючи прозорий та атомарний потік даних.

## Координація через git

Механізми координації між AI-агентами гарантують атомарність та цілісність завдань через концепцію "авторитетного claims". Система відстежує стан через "run refs" і забезпечує надійну передачу роботи між хостами за допомогою "checkpoint-handoff". Усі результати публікуються через захищений процес "fenced publish" у гілку `main`.
