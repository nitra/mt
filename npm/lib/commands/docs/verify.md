## Огляд

Огляд: Файл виконує структурну перевірку, перевіряючи наявність та зміст файлу `fact_NNN.md` у поточній дирекції. Він призначений для забезпечення цілісності конфігураційних фактів шляхом перевірки їх існування та надання підтвердження через контекст.

Поведінка

1. Ініціалізація сесії.
2. Визначення поточного робочого каталогу.
3. Отримання даних фактів з директорії.
4. Перевірка існування та наявності `fact_NNN.md`.
5. Якщо факт не знайдено або порожній, повернення помилки.
6. Якщо факт знайдено, отримання тексту з `fact_NNN.md`.
7. Отримання даних з `task.md` для генерації звіту про завершення.
8. Виведення структури перевірки та контексту для самооцінки агента.
9. Повернення коду, що вказує на структурну відповідність.
