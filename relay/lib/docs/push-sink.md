## Огляд

Dev-sink push-доставки відповідає за накопичення нотифікацій у пам'яті (подібно до використання magic tokens в auth). Він створює записи про повідомлення та збирає їх для подальшої доставки. Реальний FCM-транспорт підключається до цього інтерфейсу через метод `deliver` окремою задачею. Модуль DevPushSink є read-only і не здійснює запис у файлову систему чи базу даних.

## Поведінка

DevPushSink
1. Створюється екземпляр `DevPushSink` для ініціалізації внутрішнього списку доставлень.
2. Для передачі повідомлення в системі викликається метод `deliver` з назвою акаунта-отримувача та змістом повідомлення.
3. У цьому методі створюється запис про доставлення, який включає ідентифікатор акаунта, основний шлях, причину та опціональний посилання, і додається до внутрішнього списку `this.deliveries`.

## Публічний API

DevPushSink — реалізує механізм для відправки даних у середовищі розробки.
